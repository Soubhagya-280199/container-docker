Container :
-> Containers are lightweight, portable, and self-sufficient units that package an application along with its dependencies, libraries, and configurations, ensuring    consistent behavior across different environments.
-> It is lighweight in nature that  means it do not reqquire full OS to run any aplications. It uses resources from the base OS on which the container is running.

Docker: 
-> The most widely used container platform.
-> Docker provides user interface for containerization. Which enables user to provide all the dependecies list in Dockerfile then it will create docker image, from image we will create Docker container.
-> All the above functionality runs and executed in Docker Engine & command line.
